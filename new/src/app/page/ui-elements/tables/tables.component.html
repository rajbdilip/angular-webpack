Tables

Tables are very challenging on mobile platforms due to the very constrained screen real estate (usually no more than 320px). Therefor content authors should strive to avoid tables if possible and layout the information in an alternative way. If this cannot be avoided, there are some rough guidelines around content that goes into tables. Please note that this guidelines are not 100% hard constrains, but should rather serve as a general direction.

  • 3 columns recommended, max 4 columns, more will not fit horizontally on a mobile screen
  • 1-4 rows recommended, more will introduce a lot of scrolling on mobile devices
  • 2-3 words per column header, more will wrap on mobile screen and make for a unpleasant layout
  • 1-8 words per table cell content, try to avoid long words to enable mobile devices to wrap properly

It is highly recommended to test the table layout on mobile devices before deploying for production and check if the full table will be displayed properly.
